# QMS-Flow (Quality Management System)

> A centralized, web-based platform designed to digitize and automate ISO 9001:2015 compliance processes for Company XYZ Sdn. Bhd.

## 📌 Project Overview
QMS-Flow replaces manual, paper-based workflows with an automated digital platform. It is designed to help organizations achieve and maintain **ISO 9001:2015 compliance** efficiently. The system serves various stakeholders, including Quality Managers, Internal Auditors, Department Heads, and General Employees, providing a single source of truth for all quality-related operations.

**Course:** SECJ2203 - Software Engineering (Universiti Teknologi Malaysia)  
**Team:** Innovating Solutions (QualiTech Solutions)

## ✨ Key Features & Modules
The system is divided into four core modules to handle different aspects of quality management:

* **🔍 Audit Management** * Schedule, conduct, and track internal/external audits.
    * Maintain strict regulatory constraints (e.g., 7-year audit log retention).
* **📄 Document Control** * Centralized repository for Standard Operating Procedures (SOPs).
    * Version control and departmental approval workflows.
* **🗣️ Customer Complaints** * Portal for logging operational and customer complaints.
    * Automated routing to appropriate department heads.
* **🛠️ Corrective Actions (CAPA)**
    * Initiate Corrective and Preventive Actions from audit findings or complaints.
    * Strict data validation (e.g., Source ID verification) to ensure data integrity.

## 👥 User Roles (Personas)
* **Quality Manager (e.g., Amina):** Oversees the entire compliance workflow, audits, and CAPA.
* **Internal Auditor:** Conducts inspections and logs non-conformities.
* **Department Head:** Approves document changes and monitors departmental compliance.
* **Employees:** Accesses SOPs and logs daily operational issues.

## 🚀 Development Methodology
This project was developed using the **Agile Scrum Methodology**. 
* **System Requirements Specification (SRS):** Built solid product baselines translating business needs into detailed User Stories with strict acceptance criteria.
* **Sprints:** Features were organized into a Product Backlog and prioritized across multiple Sprints (e.g., prioritizing Audit Management in early sprints, followed by CAPA integration).
